Mantenedor de accesos a proyectos

Gestiona los usuarios y contraseñas que pueden abrir cada proyecto restringido de tu sitio, y genera el archivo cifrado.

Todo ocurre en tu navegador — nada se sube a internet. No requiere instalar nada.

Cómo funciona el acceso: el HTML de cada proyecto se cifra (AES-256). Cada usuario que agregues aquí podrá abrirlo con su contraseña; el contenido se descifra dentro del navegador de quien ingresa. Para agregar o quitar usuarios de un proyecto existente, vuelve a cargar aquí el mismo HTML con la lista de usuarios actualizada y reemplaza el archivo .enc.json.

1 Datos del proyecto

El título y la categoría se muestran en la tarjeta pública (sin secretos).

2 Contenido HTML del proyecto

Sube el archivo .html del proyecto (o pégalo). Debe ser autónomo.

3 Usuarios y contraseñas autorizados

Cada fila es un usuario con acceso a este proyecto.

Usa contraseñas robustas. En un sitio estático cualquiera puede descargar el archivo cifrado e intentar adivinar la clave sin conexión; una contraseña larga lo hace inviable.

4 Resultado

Se descargó . Súbelo a la carpeta projects/ de tu repositorio (reemplazando el anterior si es una actualización).
